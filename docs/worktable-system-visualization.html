Work Table, Assignment, Timesheet, and Period Model

Bigdil models sold scope, planned staffing, actual work, and period closing as separate layers. The work table is the reconciliation view that groups those layers by phase, task, profile, employee, and period.

System Pipeline

From commercial baseline to frozen actuals

Sold scope

QuoteLine

Defines the budget for a task/profile pair: sold days, sell rate, assumed cost, revenue, and budget cost.

Planning

PlannedDay

Stores forecast days per project, period, task, profile, and optional employee assignment.

Execution

TimesheetEntry

Records actual employee work. Approval freezes applied cost and sell rates onto the entry.

Control

ProjectPeriodStatus

Controls period state: future, open, consolidation, or frozen. This decides whether the table shows plan or actuals.

Reporting

Snapshot

Captures immutable metrics, scope, and work rows when a period is frozen.

Period Lifecycle

How a column changes behavior

FUTURE

Planner edits forecast cells. Work table reads and writes PlannedDay rows.

OPEN

Current delivery period. Forecast remains editable while employees submit timesheets.

CONSOLIDATION

Planning is locked. Approved timesheets become the displayed work table values.

FROZEN

Period is closed. A snapshot stores immutable reporting rows and financial metrics.

Operational Flows

What happens in common workflows

Assign Employee

  1. Planner picks an employee on an unassigned work table row.
  2. API moves existing unassigned PlannedDay rows to that employee.
  3. If no real row exists yet, API seeds a zero-day PlannedDay row in the first assignable period.
  4. Frontend reloads the work table and shows the employee row.

Edit Planned Days

  1. Planner edits an employee or unassigned leaf cell.
  2. API upserts PlannedDay for the exact period, task, profile, employee key.
  3. Totals roll up to profile, task, phase, and project rows.
  4. Closed periods ignore plan and show actuals instead.

Approve Timesheet

  1. Employee submits work against project, task, profile, and period.
  2. Approver approves the TimesheetEntry.
  3. Approval freezes applied cost and sell rates.
  4. When period closes, approved actuals replace plan in the work table.
